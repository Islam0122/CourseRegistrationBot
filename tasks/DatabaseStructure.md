# Database Structure — CourseRegistrationBot

## 1️⃣ Таблица `users`

| Поле       | Тип         | Описание                                 |
|------------|------------|------------------------------------------|
| id         | INTEGER PK | Уникальный идентификатор пользователя   |
| user_id    | INTEGER    | Telegram user_id                         |
| name       | TEXT       | Имя пользователя                        |
| created_at | DATETIME   | Дата регистрации пользователя            |

**Примечания:**  
- Сохраняем каждого пользователя при первом запуске бота (`/start`).  
- Используется для связывания с таблицей `registrations`.  

---

## 2️⃣ Таблица `courses`

| Поле        | Тип         | Описание                                 |
|-------------|------------|------------------------------------------|
| id          | INTEGER PK | Уникальный идентификатор курса           |
| title       | TEXT       | Название курса                            |
| description | TEXT       | Описание курса                            |
| created_at  | DATETIME   | Дата добавления курса                     |

**Примечания:**  
- Список курсов можно обновлять через парсинг / API (`/update_courses`).  
- Используется для отображения пользователю и связывания с таблицей `registrations`.  

---

## 3️⃣ Таблица `registrations`

| Поле          | Тип         | Описание                                  |
|---------------|------------|-------------------------------------------|
| id            | INTEGER PK | Уникальный идентификатор записи           |
| user_id       | INTEGER    | Ссылка на таблицу `users`                 |
| course_id     | INTEGER    | Ссылка на таблицу `courses`               |
| registered_at | DATETIME   | Дата и время регистрации                  |

**Примечания:**  
- Проверка уникальности: один студент не может зарегистрироваться дважды на один курс.  
- Можно легко получить список студентов по курсу и список курсов по студенту.
